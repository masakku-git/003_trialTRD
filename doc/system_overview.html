Python自動実行ポイント
cron毎営業日 08:45 JST に完全自動で実行される6ステップ
1

MarketScanner 自動

ウォッチリスト全銘柄の直近2日データをyfinanceで取得。変化率・出来高でスコアリングし上位3〜5銘柄を選定。候補のみDB差分取得。

agents/market_scanner.py tools/data_fetcher.py
2

TechnicalAnalyst 自動

DBのOHLCVデータで全戦略の generate_signal() を実行。MA/RSI/MACD等を計算し、最も信頼度の高いシグナルを採用。

agents/technical_analyst.py strategies/*.py
3

BacktestValidator 自動

戦略の backtest() で統計検証。勝率≥40% / RR≥0.3 / サンプル≥5 を通過基準に。7日以内のDBキャッシュを優先。

agents/backtest_validator.py
3.5

StrategyCritic 自動

「悪魔の代弁者」フィルタ。過学習疑い・RSI矛盾等をヒューリスティック検出。reject/caution/approve の3段階判定。

agents/strategy_critic.py
4

RiskManager 自動

RR≥1.5 / ポジションサイズ計算 / スロット・キャッシュ残高チェック。ポートフォリオ全体のリスクを評価。

agents/risk_manager.py
5

Executor (注文実行) 自動

Futu OpenD経由でmoomoo証券に注文発行。ポジション・スナップショットをDBに記録。

tools/executor.py tools/futu_client.py

戦略の新規開発

  • strategies/ に新しい戦略ファイルを作成
  • BaseStrategy を継承し generate_signal() / backtest() を実装
  • ファイル追加のみで自動登録 (StrategyRegistry)
  • 例: ma_cross.py, rsi_oversold.py, breakout.py

既存戦略の改善

  • バックテスト結果の分析と閾値チューニング
  • 新たなテクニカル指標の追加
  • StrategyCriticのヒューリスティック改善
  • RiskManagerのパラメータ最適化

コードレビュー・デバッグ

  • ログ分析による問題特定
  • エラー原因の調査と修正
  • パフォーマンスボトルネックの特定
  • セキュリティレビュー

ウォッチリスト管理

  • data/watchlist.json の銘柄追加・削除
  • スクリーニング基準の見直し
  • 市場環境に応じた対象銘柄の調整

重要: Claude Codeは「開発時」のみ使用

日次の自動売買実行時にClaude APIは一切呼び出されません。Proプラン契約内のClaude Codeで戦略を開発・テストし、 完成したコードをサーバにデプロイすれば、以降はPythonだけで自動運用されます。ランニングコストはAPI利用料ゼロです。

人間のオペレーション詳細 (1/3)
初期セットアップ -- サーバ構築から本番稼働までの全手順
1

サーバ契約・OS設定 人間

Hetzner Cloud で CX22 インスタンスを作成 (Ubuntu 24.04)。SSH鍵登録・rootパスワード設定。
コンソールからログインし bash setup_server.sh を実行。ユーザー作成・ufw・JSTタイムゾーン・Python venv が一括で構築される。

2

Futu OpenD の起動と二段階認証 人間

Futu OpenD をインストールし systemd サービスとして登録。初回起動時にmoomooアプリで二段階認証が必要（人間しかできない操作）。
認証完了後、systemctl status futu_openD で接続確認。ヘッドレス運用の可否は事前にmoomooサポートへ確認が必要。

3

コードデプロイ・環境変数設定 人間

git clone でコードを配置し、pip install -r requirements.txt で依存インストール。
cp .env.example .env で設定ファイルを作成し、取引パスワード・口座ID・リスクパラメータを手動で記入（機密情報のため人間が直接入力）。

4

動作確認・ペーパートレード 人間が判断

python main.py --init-db でDB初期化 → --dry-run で動作テスト → --scan-only でスキャン確認。
最低2週間のペーパートレード結果を人間が評価してから、.env の FUTU_TRADE_ENV=REAL に切り替える判断を下す。

5

cron登録・自動運用開始 人間

crontab -l で setup_server.sh が登録した cron ジョブを確認。
毎営業日 08:45 JST に main.py が自動実行される状態になれば、初期セットアップ完了。以降は自動運用。

なぜ人間が必要か

• 機密情報（パスワード・口座ID）の入力
• moomoo二段階認証（物理端末が必要）
• ペーパートレード結果の評価と本番切替の判断
• サーバインフラの契約・管理

人間のオペレーション詳細 (2/3)
Claude Codeを使った戦略開発 -- 人間が起動・指示・レビュー・デプロイする
1

Claude Codeの起動 人間

ローカルPC (またはサーバ上) で claude コマンドを実行しClaude Codeを起動。
プロジェクトディレクトリに移動してから起動することで、コードベース全体が参照可能になる。

cd ~/bot && claude
2

戦略開発の指示 人間

自然言語で何を作りたいかをClaude Codeに伝える。人間がアイデア・方針を決め、実装はClaude Codeが行う。

指示の例:
> 「MACDクロスオーバー戦略を新規作成して。
  BaseStrategyを継承して strategies/macd_cross.py に作成」
改善の例:
> 「ログを分析して。昨日のdry-run結果を見て
  ma_cross戦略の閾値を調整して」
デバッグの例:
> 「logs/trading_20260318.log のエラーを調査して修正して」
3

Claude Code がコードを生成・修正 Claude Code

Claude Codeが strategies/*.py やエージェントファイルを読み込み、コードを生成・編集する。
人間はファイル変更の差分を確認し、承認 (Allow) または拒否する。

4

レビュー・テスト・デプロイ 人間が判断

変更内容を確認し、python main.py --dry-run でテスト。問題なければ人間が git push → サーバで git pull してデプロイ。

git add / commit / push → サーバで git pull

人間が担う役割

  • Claude Codeの起動タイミングの判断
  • 何を作る/改善するかの意思決定
  • 生成コードの差分レビューと承認
  • テスト実行と結果の評価
  • git push / デプロイの実行判断

Claude Codeが担う役割

  • コードベースの読解と理解
  • 戦略コードの生成・編集
  • BaseStrategy 準拠の実装
  • ログ解析・バグ特定
  • リファクタリング提案

  • ログファイル確認: logs/trading_YYYYMMDD.log

  • 注文が発行されたか / されなかった理由

$ tail -50 ~/bot/logs/trading_$(date +%Y%m%d).log
$ grep -i "error\|warning" ~/bot/logs/trading_*.log

週次レビュー (週末 15分)

  • portfolio_snapshots テーブルで資金推移を確認
  • 今週の注文履歴 (orders テーブル) を振り返り
  • 勝率・損益の傾向を把握
  • 戦略別パフォーマンスの比較
  • ウォッチリスト銘柄の見直し検討
$ sqlite3 ~/bot/db/trading.db
> SELECT * FROM portfolio_snapshots ORDER BY date DESC LIMIT 5;
> SELECT * FROM orders WHERE date >= date('now', '-7 days');

緊急対応 (発生時)

  • 市場急変 (暴落・ストップ安) → cron一時停止
  • Futu OpenD 切断 → 再認証・サービス再起動
  • 連続エラー → ログ確認後 Claude Code でデバッグ
  • 想定外の大量注文 → 手動でポジションクローズ
